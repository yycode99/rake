## 1. 产品概述

### 1.1 核心痛点

在大模型辅助开发场景中，需求方（产品经理、工程师）与 AI 协作时，纯文字描述需求普遍存在以下问题：

- 界面上下文缺失，AI 不知道"改哪里"
- 修改位置不明确，无法精准定位代码
- 逻辑表达抽象，AI 产出与预期偏差大
- 代码与界面对应关系难以说明

### 1.2 解决方案

高度还原「面对面讲解需求」的交互方式，支持用户以**屏幕演示 + 鼠标标注 + 自然语言描述**的形式向 AI 同步讲解需求。

工具将全程录音、记录鼠标轨迹、捕获屏幕内容、采集标注事件，并通过时序化数据结构将这四路信号精确对齐，最终通过三阶段 LLM 处理管线生成结构化需求文档，供 Claude Code、Codex 等工具直接引用。

### 1.3 适用场景

- 产品经理向 AI 讲解业务需求
- 工程师向 AI 讲解技术实现与代码结构
- 前后端对接界面与逻辑

## 4. 核心数据结构：时序化 Session

整个产品的骨架。所有采集信号以毫秒时间戳对齐到同一时间轴。

```typescript
// 基础时间单位：相对于 session 开始时刻的毫秒数
type Timestamp = number;

// ─── 四条原始数据流 ───────────────────────────────────────

/** 1. 音频流（连续录音） */
interface AudioTrack {
  startTime: Timestamp;
  buffer: ArrayBuffer;        // 完整录音，WebM Opus
  sampleRate: number;         // 用于精确时间对齐
}

/** 2. 鼠标轨迹流（10Hz 采样） */
interface MousePoint {
  t: Timestamp;
  x: number;                  // 相对于选定屏幕左上角的坐标
  y: number;
  isDown: boolean;            // 是否按下（绘制中）
}
type MouseTrack = MousePoint[];

/** 3. 屏幕帧流（变化触发 + 定时兜底，~1fps） */
interface ScreenFrame {
  t: Timestamp;
  imageData: string;          // base64 PNG，压缩至 1280px 长边
  triggerReason: 'periodic' | 'user_action' | 'annotation_change';
}
type ScreenFrameTrack = ScreenFrame[];

/** 4. 标注事件流（离散事件） */
type AnnotationType = 'rect' | 'circle' | 'freehand' | 'arrow' | 'text';

type AnnotationEvent =
  | { t: Timestamp; op: 'add';    id: string; type: AnnotationType; data: AnnotationData }
  | { t: Timestamp; op: 'remove'; id: string }
  | { t: Timestamp; op: 'clear' };

interface AnnotationData {
  points: { x: number; y: number }[];  // 统一点集描述所有形状
  color: string;
  label?: string;                       // 文字贴纸内容
}

/** 5. 语义上下文轨道（扩展采集，可选） */
type SemanticSnapshot =
  | { t: Timestamp; source: 'browser'; data: DOMSnapshot }
  | { t: Timestamp; source: 'vscode';  data: VSCodeSnapshot }
  | { t: Timestamp; source: 'native';  data: NativeControlSnapshot }
  | { t: Timestamp; source: 'unknown'; data: null };

// ─── Session 完整结构 ─────────────────────────────────────

interface RecordingSession {
  id: string;
  createdAt: number;            // Unix timestamp
  status: 'recording' | 'processing' | 'done' | 'error';
  selectedScreen: SelectedScreen;

  // 五条原始轨道
  audio: AudioTrack;
  mouseTrack: MouseTrack;
  screenFrames: ScreenFrameTrack;
  annotationEvents: AnnotationEvent[];
  semanticTrack: SemanticSnapshot[];

  // 派生数据（处理后填入）
  transcript?: TranscriptResult;
  keyMoments?: KeyMoment[];
}

// ─── 派生数据结构 ─────────────────────────────────────────

/** STT 转录结果（词级时间戳） */
interface TranscriptResult {
  fullText: string;
  words: Array<{
    word: string;
    startTime: Timestamp;       // 毫秒
    endTime: Timestamp;
    confidence: number;
  }>;
  segments: Array<{             // 句子级分段
    text: string;
    startTime: Timestamp;
    endTime: Timestamp;
  }>;
}

/** 关键时刻：LLM 从转录中识别的语义锚点 */
interface KeyMoment {
  t: Timestamp;                 // 最终锚点（鼠标静止点修正后）
  tRaw: Timestamp;              // 指代词原始时间戳
  triggerPhrase: string;        // 触发词原文
  intent: string;               // 意图摘要（15 字内）
  type: 'deictic' | 'element' | 'action';
  speechContext: string;        // 前后 2.5s 的转录文本
  scene: ComposedScene;         // 此时刻的合成现场
}

/** 某一时刻的合成现场 */
interface ComposedScene {
  t: Timestamp;
  screenshot: string;           // 最近一帧截图（base64）
  activeAnnotations: AnnotationData[];  // 此刻存活的所有标注
  mousePosition: { x: number; y: number } | null;
  semanticContext: SemanticSnapshot | null;  // 最近的语义快照
  compositeImage: string;       // 合成图：截图+标注+鼠标聚光圈（base64）
}
```

## 5. 模块设计：屏幕采集

### 5.1 采集优先级

| 优先级 | 方式 | 实现 |
|--------|------|------|
| P0 | 静态截图（主力） | desktopCapturer + 帧差去重 |
| P1 | 录屏片段（≤30s） | MediaRecorder + ffmpeg.wasm 关键帧提取 |
| P2 | 实时流 | V1 不做，以定时快照替代 |
| P3 | 手动上传 | 文件选择器 / 拖拽 |

### 5.2 截图流程

```typescript
// Main Process
async function captureFrame(sourceId: string): Promise<string> {
  const sources = await desktopCapturer.getSources({
    types: ['screen'],
    thumbnailSize: { width: 1280, height: 800 }
  });
  const source = sources.find(s => s.id === sourceId);
  return source.thumbnail.toDataURL();
}
```

### 5.3 帧差去重（增量存储）

相邻帧差异低于阈值时不存储新帧，仅记录时间戳引用：

```typescript
type FrameEntry =
  | { t: Timestamp; type: 'keyframe'; imageData: string }
  | { t: Timestamp; type: 'ref'; refT: Timestamp };

async function shouldStoreFrame(newFrame: string, lastFrame: string): Promise<boolean> {
  const diff = await pixelDiff(newFrame, lastFrame, { scale: 64 });
  return diff > 0.05; // 差异超过 5% 才存新帧
}
```

实测一般屏幕操作去重后帧数量减少 60~80%。

### 5.4 三级自适应采集策略

```typescript
function adaptCaptureConfig(): CaptureConfig {
  const cpu = process.getCPUUsage().percentCPUUsage;
  const mem = process.memoryUsage().heapUsed / 1024 / 1024;

  if (cpu < 40 && mem < 500) {
    return { fps: 2, resolution: 1280, quality: 0.85 }; // 正常
  } else if (cpu < 70 && mem < 800) {
    return { fps: 1, resolution: 960,  quality: 0.75 }; // 降级
  } else {
    return { fps: 0.5, resolution: 720, quality: 0.65 }; // 最低
  }
}
```

## 6. 模块设计：覆盖层与实时标注

### 6.1 覆盖层窗口创建

覆盖层是一个全屏透明的 Electron BrowserWindow，始终置顶于其他所有程序之上：

```typescript
// Main Process
function createOverlayWindow(screen: SelectedScreen): BrowserWindow {
  const overlay = new BrowserWindow({
    x: screen.bounds.x,
    y: screen.bounds.y,
    width: screen.bounds.width,
    height: screen.bounds.height,
    transparent: true,
    frame: false,
    alwaysOnTop: true,
    skipTaskbar: true,
    focusable: false,
    hasShadow: false,
    webPreferences: {
      preload: path.join(__dirname, 'overlay-preload.js'),
    },
  });

  // 置顶级别：macOS 使用 screen-saver 最高级覆盖全屏 App
  overlay.setAlwaysOnTop(true,
    process.platform === 'darwin' ? 'screen-saver' : 'pop-up-menu'
  );

  // macOS 多桌面 / 全屏 App 支持
  if (process.platform === 'darwin') {
    overlay.setVisibleOnAllWorkspaces(true, { visibleOnFullScreen: true });
  }

  // 初始状态：完全穿透，不拦截鼠标事件
  overlay.setIgnoreMouseEvents(true, { forward: true });

  return overlay;
}
```

### 6.2 标注模式切换

用户按住 Ctrl/Cmd 键进入标注模式，松开后退出：

```typescript
// Main Process：全局快捷键监听 Ctrl/Cmd
globalShortcut.register('CommandOrControl', () => {
  overlay.setIgnoreMouseEvents(false);  // 开始拦截鼠标
  overlay.setFocusable(true);
  overlay.focus();
  overlay.webContents.send('annotation-mode-enter');
});

// Renderer（覆盖层）：监听键盘抬起
window.addEventListener('keyup', (e) => {
  if (e.key === 'Control' || e.key === 'Meta') {
    ipcRenderer.send('annotation-mode-exit');
  }
});

// Main Process：收到退出信号
ipcMain.on('annotation-mode-exit', () => {
  overlay.setIgnoreMouseEvents(true, { forward: true });
  overlay.setFocusable(false);
  overlay.webContents.send('annotation-mode-exit');
});
```

**状态说明：**

```
平时（录制中，正常操作）：
  → setIgnoreMouseEvents(true)：完全穿透，用户正常操作底层程序
  → 覆盖层仅展示已画好的标注（半透明）

按住 Ctrl/Cmd（标注模式）：
  → setIgnoreMouseEvents(false)：覆盖层接管鼠标
  → Fabric.js 接收拖拽，进行标注绘制

松开 Ctrl/Cmd（退出标注模式）：
  → 恢复穿透，标注保留显示
```

### 6.6 标注合成图导出

```typescript
// 离屏合成：截图 + 标注 + 鼠标聚光圈 → 单张 PNG
async function renderComposite(scene: {
  background: string;
  annotations: AnnotationData[];
  mousePos: { x: number; y: number } | null;
}): Promise<string> {
  const canvas = new OffscreenCanvas(1280, 800);
  const ctx = canvas.getContext('2d')!;

  // 1. 绘制背景截图
  const img = await loadImage(scene.background);
  ctx.drawImage(img, 0, 0, 1280, 800);

  // 2. 绘制标注
  for (const ann of scene.annotations) {
    renderAnnotation(ctx, ann);
  }

  // 3. 绘制鼠标聚光圈
  if (scene.mousePos) {
    renderMouseHighlight(ctx, scene.mousePos.x, scene.mousePos.y);
  }

  const blob = await canvas.convertToBlob({ type: 'image/png', quality: 0.9 });
  return blobToBase64(blob);
}

function renderMouseHighlight(ctx: CanvasRenderingContext2D, x: number, y: number) {
  // 外圈：半透明遮罩（聚焦效果）
  const mask = ctx.createRadialGradient(x, y, 40, x, y, 200);
  mask.addColorStop(0, 'rgba(0,0,0,0)');
  mask.addColorStop(1, 'rgba(0,0,0,0.45)');
  ctx.fillStyle = mask;
  ctx.fillRect(0, 0, ctx.canvas.width, ctx.canvas.height);

  // 内圈：高亮光晕
  ctx.beginPath();
  ctx.arc(x, y, 18, 0, Math.PI * 2);
  ctx.strokeStyle = '#FFEB3B';
  ctx.lineWidth = 3;
  ctx.stroke();

  // 十字准星
  ctx.strokeStyle = '#FF4444';
  ctx.lineWidth = 1.5;
  ctx.beginPath();
  ctx.moveTo(x - 12, y); ctx.lineTo(x + 12, y);
  ctx.moveTo(x, y - 12); ctx.lineTo(x, y + 12);
  ctx.stroke();
}
```

```typescript
  connect() {
    try {
      this.ws = new WebSocket('ws://localhost:27513');
      this.ws.onopen = () => { this.retryCount = 0; this.sendHandshake(); };
      this.ws.onclose = () => {
        if (this.retryCount < this.MAX_RETRIES) {
          // 指数退避：1s, 2s, 4s, 8s, 16s
          setTimeout(() => this.connect(), Math.pow(2, this.retryCount++) * 1000);
        }
        // 超过最大重试次数，静默放弃
      };
      this.ws.onerror = () => { /* 静默失败 */ };
    } catch { /* 静默失败 */ }
  }
}
```

## 8. 模块设计：三次 LLM 交互

录制结束后，数据经过三次 LLM 交互处理，共三个阶段：

```
   │
   ▼
```

## 9. 模块设计：时间锚点定位

这是整个系统最核心的技术难点：**如何从语义找到精确的时间点**。

### 9.1 核心原理

Whisper 输出词级时间戳，精度 ±200ms。关键时刻的精确定位 = **指代词时间戳 + 鼠标静止点交叉验证**。

```
「关键时刻」≠「触发词出现的时刻」

例：
00:02 「我们先看一下整体布局」  ← 鼠标在乱动
00:05 「然后这里...」           ← 鼠标移到导航栏  ← 真实指代时刻
00:06 「这个导航栏」            ← 语义最清晰，但鼠标开始离开
00:07 「我希望它固定在顶部」

最终锚点 = 00:05（鼠标静止点），而非 00:06（指代词）
```

### 9.2 双信号交叉验证算法

```typescript
function findAnchorPoint(
  phrase: { t: Timestamp },
  mouseTrack: MousePoint[]
): Timestamp {
  // 在指代词前后 1.5s 窗口内找鼠标静止点
  const windowStart = phrase.t - 1000;
  const windowEnd   = phrase.t + 500;
  const window = mouseTrack.filter(p => p.t >= windowStart && p.t <= windowEnd);

  // 滑动窗口：找连续 300ms 内位移 < 15px 的区间
  for (let i = 0; i < window.length; i++) {
    const segment = window.filter(p => p.t - window[i].t <= 300);
    const maxDrift = Math.max(...segment.map(p =>
      Math.hypot(p.x - window[i].x, p.y - window[i].y)
    ));
    if (maxDrift < 15) {
      // 找到静止区间，取中点时刻
      return segment[Math.floor(segment.length / 2)].t;
    }
  }

  // 没找到静止点，fallback 到指代词自身时刻
  return phrase.t;
}
```

### 9.3 时序重建：给定 t* 合成 ComposedScene

```typescript
function composeScene(t: Timestamp, session: RecordingSession): ComposedScene {
  // 1. 找 t 之前最近的屏幕帧
  const frame = findLatestBefore(session.screenFrames, t);

  // 2. Replay 标注事件流至 t 时刻
  const activeAnnotations = replayAnnotations(session.annotationEvents, t);

  // 3. 插值鼠标位置
  const mouse = interpolateMouse(session.mouseTrack, t);

  // 4. 找最近的语义快照
  const semantic = findLatestBefore(session.semanticTrack, t);

  // 5. 离屏 Canvas 合成
  const compositeImage = renderComposite({ background: frame.imageData, activeAnnotations, mousePos: mouse });

  return { t, screenshot: frame.imageData, activeAnnotations, mousePosition: mouse, semanticContext: semantic, compositeImage };
}

function replayAnnotations(events: AnnotationEvent[], upTo: Timestamp): AnnotationData[] {
  const active = new Map<string, AnnotationData>();
  for (const e of events) {
    if (e.t > upTo) break;
    if (e.op === 'add')    active.set(e.id, e.data);
    if (e.op === 'remove') active.delete(e.id);
    if (e.op === 'clear')  active.clear();
  }
  return [...active.values()];
}
```

### 9.4 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 鼠标没动（界面元素词描述） | 仅用指代词时间戳，第 3 次 LLM 自行在图中定位元素 |
| 指代词与鼠标静止点时间差 > 2s | 保留两个时刻，各生成一张合成图，第 3 次 LLM 自行判断 |
| 连续密集指代（快速切换多区域） | 降级为每个指代词时刻对应鼠标位置（不要求静止） |

### 9.5 定位精度汇总

| 定位方式 | 适用场景 | 精度 |
|---------|---------|------|
| 指代词时间戳 + 鼠标静止点交叉 | 正常讲解（最常见） | ±300ms |
| 仅指代词时间戳 | 鼠标没动或快速移动 | ±500ms |
| 界面元素词 + LLM 视觉搜索 | 无指示词，靠内容描述 | 依赖 LLM 视觉理解 |

### 10.3 历史关键帧检索

追问时如何找到相关历史帧，采用**方案一 + 方案三组合**：

**方案一：关键词匹配（默认）**

```typescript
function findRelevantFrames(query: string, keyMoments: KeyMoment[]): KeyMoment[] {
  const queryTokens = segment(query); // 分词
  return keyMoments
    .map(km => ({
      km,
      score: queryTokens.filter(t => km.intent.includes(t) || km.phrase.includes(t)).length
    }))
    .filter(({ score }) => score > 0)
    .sort((a, b) => b.score - a.score)
    .map(({ km }) => km);
}
```

**方案三：LLM 决策（匹配失败时的兜底）**

```
发给 LLM（纯文本）：
  上一轮 PRD + 所有关键时刻文字摘要 + 用户追问
  → 请问你需要查看哪些时刻的截图？输出 JSON: { "needed": [1, 2] }

按返回编号拉取对应关键帧图，发起正式回答。
```

```typescript
  sourceId: string;       // desktopCapturer source id
  displayId: number;      // Electron display id
  bounds: Rectangle;      // 屏幕位置和尺寸
  scaleFactor: number;    // 高分屏缩放比例（Retina 为 2）
}

function applyScreenSelection(selected: SelectedScreen) {
  // 覆盖层移动到选定屏幕并铺满
  overlayWindow.setBounds(selected.bounds);

  // 截图时只截选定的 source
  captureConfig.sourceId = selected.sourceId;

  // 鼠标坐标转换为相对于选定屏幕的坐标
  mouseTracker.setOrigin({ x: selected.bounds.x, y: selected.bounds.y });
}
```

**注意：** 高分屏（Retina，scaleFactor = 2）时，截图实际像素是逻辑像素的 4 倍。鼠标坐标标注时需乘以 `scaleFactor` 才能与截图像素对齐。

## 14. 工程保障

### 14.1 性能与资源占用

**压力来源：** 屏幕截图是唯一主要压力（5~15% CPU），其他采集流几乎无感。

**三项保障措施：**

1. 三级自适应截图配置（见 5.4）
2. 帧差去重减少 60~80% 存储（见 5.3）
3. 异步写入队列，不阻塞主线程：

```typescript
class WriteQueue {
  private queue: Array<() => Promise<void>> = [];
  private running = false;

  push(task: () => Promise<void>) {
    this.queue.push(task);
    if (!this.running) this.drain();
  }

  private async drain() {
    this.running = true;
    while (this.queue.length > 0) {
      await this.queue.shift()!();
    }
    this.running = false;
  }
}
```

**10 分钟录制数据量估算（去重后）：**

| 数据类型 | 估算大小 |
|---------|---------|
| 音频（WebM Opus） | ~8MB |
| 鼠标轨迹 | ~120KB |
| 屏幕帧（去重后） | ~18MB |
| 标注事件 | <10KB |
| 语义快照 | <50KB |
| **合计** | **~26MB** |

### 14.2 录制时长限制

```
软限制（录制 > 15 分钟时提示）：
  「录制已超过 15 分钟，建议分段以获得更准确的需求分析」

硬限制（默认 30 分钟，可配置关闭）：
  超出后自动暂停，提示用户保存并开始新录制
```

超过 10 分钟自动分段，发给 LLM 时均匀采样不超过 20 张关键帧：

```typescript
function mergeSegmentsForLLM(segments: RecordingSession[]): KeyMoment[] {
  const allMoments = segments.flatMap(s => s.keyMoments ?? []);
  return subsample(allMoments, 20); // 均匀采样，最多 20 帧
```

### 14.3 错误处理与恢复

**场景 A：录制中途崩溃**

使用 SQLite WAL 模式，边录边持久化：

```typescript

// 应用重启后自动检测并提示恢复
function recoverUnfinishedSession(): RecordingSession | null {
  const unfinished = db.prepare(
    `SELECT * FROM sessions WHERE status = 'recording'`
  ).get();
  if (unfinished) return repairSession(unfinished); // 截断损坏尾部
  return null;
}
```

**场景 B：LLM 调用失败**

三级重试 + 降级：

```typescript
async function callLLMWithRetry<T>(fn: () => Promise<T>, fallback: () => T): Promise<T> {
  const delays = [1000, 3000, 8000];
  for (let i = 0; i < delays.length; i++) {
    try {
      return await fn();
    } catch (err) {
      if (isRateLimitError(err)) { await sleep(delays[i]); continue; }
      if (isAuthError(err)) throw new UserFacingError('API Key 无效，请在设置中检查');
      if (i === delays.length - 1) return fallback(); // 三次失败后降级
    }
  }
}
```

**场景 C：转录质量差**

```typescript
function assessTranscriptQuality(transcript: TranscriptResult): QualityReport {
  const avgConfidence = transcript.words.reduce(
    (sum, w) => sum + w.confidence, 0
  ) / transcript.words.length;

  return {
    isAcceptable: avgConfidence > 0.75,
    suggestion: avgConfidence < 0.75
      ? '转录质量较低，建议在安静环境重新录制，或手动编辑转录文本'
      : null
  };
```

### 14.4 本地 vs 云端边界

```
本地处理（无需网络）：
├── 屏幕截图采集        desktopCapturer
├── 音频录制            MediaRecorder
├── 鼠标轨迹采集        纯内存
├── 标注事件记录        Fabric.js + Zustand
├── 语义快照采集        WebSocket 本地服务
├── 帧差比较去重        Canvas 像素差
├── 鼠标静止点检测      纯算法
├── 标注事件 replay     纯算法
├── 离屏 Canvas 合成    OffscreenCanvas
├── 转录文本格式化      纯字符串处理
├── 关键词匹配检索      字符串匹配
└── 会话持久化          SQLite

云端处理（需要网络）：
├── 语音转文字          Whisper API（可换本地 whisper.cpp）
├── 指代词提取          LLM 第 2 次调用
└── 需求理解与输出      LLM 第 3 次调用

离线降级：
├── 录制 ✓             全部本地
├── 标注 ✓             全部本地
├── 合成图 ✓           全部本地
├── STT ✗              提示「网络恢复后处理」
└── LLM 分析 ✗         队列化，网络恢复后自动触发
```

### 14.5 API Key 安全

```
用户输入 API Key
      ↓
Renderer → IPC → Main Process（Key 不在 Renderer 内存停留）
      ↓
safeStorage.encryptString()（系统级 KeyChain 加密写盘）
      ↓
LLM 调用时 Main Process 内存中短暂存在，调用完立即 GC
      ↓
DevTools 不可见（contextBridge 不暴露 Key 相关 API）
```

生产环境禁用 DevTools：

```typescript
if (app.isPackaged) {
  mainWindow.webContents.on('devtools-opened', () => {
    mainWindow.webContents.closeDevTools();
  });
}
```
